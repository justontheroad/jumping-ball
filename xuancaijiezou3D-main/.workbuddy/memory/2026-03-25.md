## 加速背景粒子系统（螺旋隧道，跟随球颜色）
- 8000 粒子，螺旋隧道分布（3 条螺旋臂结构）
- 粒子沿 Z 轴飞掠（向球飞来），飞过后循环回远处
- 70% 极小粒子(0.06~0.18)、22% 中粒子(0.2~0.6)、8% 大光斑(0.6~1.6)
- 隧道收束效果：半径随 Z 位置变化，中间宽两头窄
- **颜色跟随球颜色**：`setBoostParticleColor(colorKey)` 设置目标颜色
  - Shader uniform `uColor` 控制所有粒子颜色，省去逐粒子更新
  - `boostParticleColorCurrent.lerp(target)` 每帧平滑过渡（速度 `BOOST_COLOR_LERP_SPEED=3.0`）
  - 球变色时（`onLandUpdateVelZ`、`resetGame`、初始化）同步调用 `setBoostParticleColor`
  - 加速开始时从当前球颜色快照，避免滞后

## 加速轨道（类型4）— 白色发光长直轨道
- `createType4(z)`：使用 `longGeo`（和长直轨道同形状），白色发光材质 `makeBoostMat()`
- 15% 概率在 `createRandomGroup` 中随机生成
- 碰到后：`colorKey: 'boost'`，不改变球颜色，不致死，不进入异色检测
- **蓄力机制**：第一次踩到加速轨道 → 原地弹一跳（velZ=0），落回同一轨道后再出发
  - `boostCharging` 标志区分首次落地 vs 蓄力回弹
  - 首次：`boostCharging=true, velZ=0`，重力切换但球不前进
  - 回弹：`boostCharging=false, onLandUpdateVelZ()`，用新周期计算前进速度
- `velY = Math.sqrt(2 * |currentGravityY| * INIT_HEIGHT)` 确保弹跳高度正确
- `recalcVelZ()`：纯重算 velZ（不改颜色），用于重力变化后同步速度
- 10秒后 `boostTimer` 归零，`currentGravityY` 恢复 `GRAVITY_Y`，同时调用 `recalcVelZ()` 同步水平速度
- 着陆特效 `triggerLandEffect` 对 'boost' 使用白色 `[220,220,255]`
- 碰撞检测 `findCollisionWithAnyGroup` 对 type4 使用 `LONG_SIZE_X/Z`
- `resetGame` 中重置 `boostTimer=0, currentGravityY=GRAVITY_Y, boostCharging=false`

## 鼠标 X 轴缓动控制
- `ballXTarget` 作为缓动目标，鼠标驱动目标位置
- `ballX` 用指数平滑（lerp）追赶目标：`ballX += (target - ballX) * factor`
- `LERP_SMOOTHNESS = 0.12`，帧率无关：`factor = 1 - (1 - 0.12)^(dt * 60)`
- 目标范围限制 ±15，`resetGame` 同步重置 `ballXTarget=0`

## 加速恢复后 velZ 不同步 bug 修复
- **问题**：加速到期恢复原始重力时，弹跳周期变长（1倍 vs 3倍），但 velZ 没有重算，导致球飞过头无法落到下一个轨道
- **修复**：新增 `recalcVelZ()` 函数，在 `boostTimer <= 0` 时调用，用新的（更长的）弹跳周期重算 velZ
